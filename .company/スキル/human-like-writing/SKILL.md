# Human-Like Writing — AIっぽさを消す前提スキル

**重要度: 最優先（前提条件）**  
あらゆる文章生成タスクで、このスキルを**最初に**参照すること。content-creator、x-post-creator、content-design などの具体的スキルは、この土台の上に乗る。

---

## なぜAIの文章に違和感が出るのか — 6つの正体

違和感の正体は全部言語化できる。全部まとめると「人間味がない」の一言に集約される。

| # | 違和感の正体 | 読者の本音 |
|---|-------------|-----------|
| ❶ | **抽象的すぎる** — 「多大な努力が必要です」みたいなふわっとした表現 | 「で、具体的には？」 |
| ❷ | **感情がゼロ** — 事実を淡々と並べるだけ | 「この人、何も感じてないの？」 |
| ❸ | **文脈が弱い** — 情報が唐突に並ぶ | 「なんで今この話？」 |
| ❹ | **論理的すぎる** — 完璧に整然としてる | 人間ってもっと雑なんですよ |
| ❺ | **単調なリズム** — 同じ構文が延々と続く | 3行目くらいで「機械的だな」ってバレる |
| ❻ | **不自然な比喩** — 「まるで〇〇のように」の連発 | 言わされてる感がすごい |

**逆に言えば、この6つを潰せばAIが書いたように見えなくなる。**

---

## 5つのルール — AIへの指示として徹底する

### ルール❶ 超具体的に書かせる

- **禁止**: 「多大な努力の末、成功した」
- **OK**: 「6ヶ月間、毎日3時間。最初の売上はたったの500円だった」
- **基準**: 読者が映像として浮かべられるかどうか

### ルール❷ 友達に話すトーンにする

- **禁止**: 「その経験は私に重要な気づきを与えてくれました」
- **OK**: 「正直あの時はガチで凹んだ」
- **効果**: このトーンだけで人が書いてると感じさせられる

### ルール❸ 感情をストレートに出す

- 事実の羅列で終わらせない
- 「嬉しかった」「悔しくて泣いた」「驚いた」— 感情の言葉を意図的に入れる

### ルール❹ ストーリーで語らせる

- **禁止**: 「箇条書きで整理して」
- **OK**: 「体験談として語って」
- **効果**: 導入→展開→結論の物語構成にするだけで全く別物になる

### ルール❺ 脱線とユーモアを許可する

- AIは基本的に真面目すぎる
- 「ちょっとした余談や軽いユーモアを入れていい」と明示的に許可する
- これが意外と効く

## チェックリスト — 生成後に必ず通す

- [ ] 抽象的でふわっとした表現がないか？（❶）
- [ ] 感情の言葉が入っているか？（❷❸）
- [ ] 文脈の流れが自然か？唐突な情報の羅列になっていないか？（❸）
- [ ] 論理的に整いすぎていないか？人間の雑さがあるか？（❹）
- [ ] 構文が単調に繰り返されていないか？（❺）
- [ ] 「まるで〇〇のように」などの不自然な比喩が連発していないか？（❻）
- [ ] 具体的な数字・シーン・体験が入っているか？（ルール❶）
- [ ] 友達に話すトーンになっているか？（ルール❷）
- [ ] ストーリー構成になっているか？（ルール❹）
- [ ] 脱線やユーモアの余地があるか？（ルール❺）
