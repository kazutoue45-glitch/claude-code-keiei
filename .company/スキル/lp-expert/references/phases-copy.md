# Fase 3 — Copy (Detalhamento)

## Explicação para o usuário

> "A copy é o texto que convence o visitante a agir. Não é texto bonito — é texto estratégico. Cada frase tem uma função: gerar identificação, quebrar objeção ou levar ao clique. Uma boa LP com copy ruim não converte. Uma LP simples com copy certeira, sim."

## Três caminhos possíveis

### Caminho A — Usuário tem copy pronta

1. Skill lê o documento fornecido
2. Cruza com as keywords da Fase 2
3. Analisa e apresenta sugestões de melhoria:
   > "Sua copy está boa, mas identifiquei X pontos que podem melhorar a conversão e o SEO:"
   > - Headline não contém a keyword primária
   > - FAQ tem poucas perguntas, recomendo expandir para 10+ (para aparecer no Google)
   > - Faltam números concretos na prova social
   > - Nenhuma citação técnica/legal (importante para IA citar sua página)
4. Pergunta: *"Deseja que eu aplique essas melhorias ou prefere manter como está?"*

### Caminho B — Usuário não tem copy

1. Delega para `/copywriting` com:
   - Briefing completo da Fase 1
   - Keywords da Fase 2
   - Análise de concorrentes (se houver URLs)
   - Nicho + objetivo da Fase 0
2. `/copywriting` gera copy completa seguindo a estrutura de seções (abaixo)
3. Apresenta a copy completa para aprovação

### Caminho C — Usuário tem copy parcial

1. Skill identifica o que existe e o que falta
2. Apresenta: *"Você tem hero, problema e FAQ. Faltam: trust bar, como funciona, depoimentos e CTA final. Deseja que eu complete?"*
3. Gera apenas as seções faltantes, mantendo o tom do que já existe

## Estrutura de Seções

```
HERO
├── Badge (categoria do serviço)
├── Headline (dor principal + keyword primária)
├── Subtítulo (prova — dado ou jurisprudência)
├── Parágrafo de contexto (expansão da dor)
└── CTA primário + microcopy de confiança

TRUST BAR
└── 4 números de credibilidade com ícones

PROBLEMA (o que o cliente não sabe)
├── Título emocional
├── Subtítulo explicativo
└── 3 cards (dor 1, dor 2, dor 3)

PROVA (o que já foi resolvido/comprovado)
├── Título com dado concreto
├── Parágrafo com citação técnica/legal
└── 4 cards com benefícios comprovados

COMO FUNCIONA
├── Título + subtítulo
└── 4 etapas numeradas (processo do serviço)

POR QUE NÓS (diferenciais + depoimentos)
├── Título provocativo
├── 3 diferenciais com ícone + texto
└── 4 depoimentos (nome + cidade + texto)

FAQ
├── Título
└── 10+ perguntas otimizadas para AEO
    (inclui keywords secundárias + long-tail)

CHECKLIST DE QUALIFICAÇÃO (se aplicável)
├── Título
└── 4 itens "Se você [situação], vale analisar"

CTA FINAL
├── Headline de urgência
├── Parágrafo de reforço
└── Botão + selos de confiança

FOOTER
└── Logo + dados da empresa + disclaimer
```
