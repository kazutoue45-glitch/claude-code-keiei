# Notion API CRUD スキル（MCP版）

## いつ使うか
- Notion DBの読み書き（ページ作成・更新・クエリ）を行うとき
- AI作業ログの記録時
- AI参照ナレッジの参照・登録時
- タスク・PJ・活動ログの参照時

## 接続方式
Notion MCP（`user-Notion` サーバー）経由で操作する。REST API直接呼び出しは不要。

## CRUD 操作パターン

### 参照（Read）

**ナレッジを検索する:**
```
notion-search: { query: "検索キーワード", data_source_url: "collection://312556a1-98c7-806d-a383-000b8400b220" }
```

**ページ内容を取得する:**
```
notion-fetch: { id: "ページURL or ページID" }
```

**DBビューをクエリする:**
```
notion-query-database-view: { view_url: "ビューURL" }
```

### 作成（Create）

**AI作業ログを記録する:**
```json
{
  "parent": { "data_source_id": "337bd381-95d2-41d5-ada5-3a69c1b64125" },
  "pages": [{
    "properties": {
      "作業名": "作業の要約タイトル",
      "作業タイプ": "実装",
      "重要度": "🟡中",
      "担当AI": "Cursor",
      "date:日時:start": "2026-03-15T10:00:00.000+09:00",
      "date:日時:is_datetime": 1,
      "作業内容": "やったことの箇条書き",
      "作業結果": "成果物・アウトプット",
      "ネクストアクション": "次にやるべきこと",
      "使用エージェント": "https://www.notion.so/参照したナレッジのページURL",
      "対象タスク": "https://www.notion.so/関連タスクのページURL",
      "プロジェクト": "https://www.notion.so/関連PJのページURL"
    }
  }]
}
```

**Sparkie Postsにポストを登録する:**

> ⚠️ **Sparkie Posts登録時は必ずNotionの `/X投稿クリエイター` スキルを参照すること。** 以下は要点の抜粋。正はNotionページ。

```json
{
  "parent": { "database_id": "30f556a1-98c7-8164-a3a2-f04b8ffd0a8b" },
  "pages": [{
    "properties": {
      "1文目フック": "ポストの1文目（タイトル）",
      "アカウント": "[\"X: @hukuro_educator\"]",
      "投稿タイプ": "通常ポスト",
      "ターゲット層": "1段目（初心者）",
      "フック種類": "[\"ストーリー\"]",
      "教育要素": "[\"ノウハウ・知識\"]",
      "投稿目的": "[\"教育\"]",
      "期待アクション": "保存",
      "画像•動画の有無": "テキストのみ",
      "準備完了": "__NO__",
      "date:予約日時:start": "2026-03-24T08:00:00.000+09:00",
      "date:予約日時:is_datetime": 1
    },
    "content": "<callout icon=\"https://www.notion.so/icons/document_gray.svg\">\nポスト本文をここに書く\n</callout>"
  }]
}
```

> ⚠️ **予約日時は必須**: 未設定だとビューに表示されない。日付指定がなければ**当日**。複数ポストでもすべて当日。
>
> ⚠️ **時間は分散させる**: 08:00〜21:30の間で間隔を空けて設定する。全ポスト同じ時間にしない。
>
> ⚠️ **本文はコールアウトブロック内に格納**: コールアウトの外に書くとSparkieが認識しない。アイコンは外部画像URL（例: `https://www.notion.so/icons/document_gray.svg`）を使用。
>
> ⚠️ **parent は `database_id` を使用**: `data_source_id` ではなく `parent.database_id` を指定する。

**プロパティの型ごとの書式:**
- **date**: `"date:プロパティ名:start"`, `"date:プロパティ名:end"`, `"date:プロパティ名:is_datetime"` の展開形式
- **select / status**: 定義済み選択肢の文字列をそのまま指定
- **relation**: ページURLの文字列（`"https://www.notion.so/xxxxx"` 形式）
- **person**: ユーザーIDの文字列
- **checkbox**: `"__YES__"` / `"__NO__"`
- **number**: JavaScriptの数値（文字列ではない）
- **url**: URL文字列

### 更新（Update）

**プロパティを更新する:**
```json
{
  "page_id": "ページID",
  "command": "update_properties",
  "properties": {
    "ステータス": "完了"
  }
}
```

**コンテンツを部分更新する:**
```json
{
  "page_id": "ページID",
  "command": "update_content",
  "content_updates": [{
    "old_str": "置換前のテキスト",
    "new_str": "置換後のテキスト"
  }]
}
```

**コンテンツを全面書き換えする:**
```json
{
  "page_id": "ページID",
  "command": "replace_content",
  "new_str": "新しいコンテンツ全文"
}
```

### 削除
- 物理削除は原則禁止
- タスクのステータスを「アーカイブ」に変更する

## AI作業ログ記録ルール

### 記録タイミング
- ユーザーに成果を報告する規模の作業を完了したとき
- 単純なQ&Aや軽微な修正は記録不要

### 必須項目
- 作業名（簡潔な要約）
- 作業タイプ（実装 / 設計 / 修正 / 調査 等）
- 担当AI = "Cursor"
- 日時（作業実施日時）
- 作業内容（やったこと）
- 作業結果（成果物）

### リレーション項目（該当するものは必ず設定する）
- **使用エージェント**（参照したAI参照ナレッジのページURL）— 作業中にDB_AI参照ナレッジのページを参照した場合、そのページURLを必ずリレーションに設定する
- **対象タスク**（関連タスクのページURL）— 作業がDB_タスク管理の特定タスクに紐づく場合
- **プロジェクト**（関連PJのページURL）— 作業がDB_プロジェクト管理の特定PJに紐づく場合
- **日報**（日報ページのURL）— 該当する日報がある場合

### その他推奨項目
- ネクストアクション
- 重要度

## リレーション構造

```
DB_ゴール設定 → DB_プロジェクト管理 → DB_タスク管理
                    ↑↓                      ↑↓
             DB_活動ログ・MTG          DB_AI作業ログ
                                            ↓
                                  DB_AI参照ナレッジ
```

自己参照: タスク（親↔小）、ナレッジ（関連）、活動ログ（関連MTG）、PJ（関連PJ）

## エラーハンドリング

- プロパティ名・選択肢が合わない → `notion-fetch` でDBを再取得しスキーマを確認
- ページが見つからない → `notion-search` で正しいIDを検索
- 書き込み権限エラー → ユーザーに確認（書き込み制限に該当する可能性）
