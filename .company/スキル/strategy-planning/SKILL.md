# 戦略立案スキル — 逆算思考の原則

## いつ使うか

- 事業戦略・マーケティング戦略・コンテンツ戦略を立案するとき
- 施策・タスクの実行順序を決めるとき
- 「これは本当に逆算になっているか？」を検証するとき
- 戦略が浅い・ズレていると感じたとき

---

## 逆算思考とは何か

### 定義

**逆算思考** = 「ゴールから出発し、そのゴールを実現するために直前に何が必要かを問い続けること」

「今できることから積み上げる」のは**純算（前進計算）**であって逆算ではない。

### 逆算と純算の違い

| | 逆算 | 純算 |
|--|------|------|
| 出発点 | ゴール | 今できること |
| 問いの方向 | 「これを達成するには直前に何が必要か？」 | 「今日何ができるか？」 |
| 施策の評価軸 | ゴールへの貢献度 | 実行のしやすさ |
| 典型的な失敗 | ゴールを忘れて手段が目的化する | ゴールとずれた行動を積み重ねる |

### 核心にある問い

逆算をするとき、各ステップで問い続ける唯一の質問：

> **「このゴールを達成するために、直前に何が必要か？」**

この問いをゴールから今日の行動まで繰り返す。これが逆算の実体。

---

## 戦略立案の手順

### Step 1: ゴールを定義する

曖昧なゴールは逆算できない。具体的・測定可能・期限付きで定義する。

- NG例: 「認知を広げたい」
- OK例: 「2026年Q2末までにLINE登録者100人を達成する」

### Step 2: ゴールの直前に何が必要かを問う

「このゴールが達成されるためには、直前に何が起きている必要があるか？」

例: LINE登録者100人 → 「LINE登録LPに十分な数の人が訪れ、登録している必要がある」

### Step 3: さらに「その直前」を問い続ける

Step 2で出てきた「必要なもの」に対して、同じ問いを繰り返す。

今日の行動レベルまで分解できたら完成。

### Step 4: 実行順序は「逆算順の逆」になる

逆算で分解した順序の**逆が実行順序**。

- 逆算順: ゴール → 中間目標A → 中間目標B → 今日の行動
- 実行順: 今日の行動 → 中間目標B → 中間目標A → ゴール

**重要**: 実行は「今日の行動」から始まるが、**設計は必ずゴールから始める**。

## コンテンツ戦略への具体的な適用

### ふくろうビジネスへの適用例

```
【ゴール】Notion BizBase Personal 購入 / 企業向け成約 / 有料note購入
    ↑ 何があれば購入するか
LINE ステップ配信（教育・信頼構築 → 自然な購買意欲）
    ↑ 何があればLINEに登録するか
LINE登録LP（魅力的な無料プレゼント + 明快な登録動線）
    ↑ 何があればLPに来るか
note記事（詳細な教育コンテンツ + 末尾にLINE登録CTA）
    ↑ 何があればnote記事を読むか
X Articles（note記事の核心を切り出した予告・興味喚起）
    ↑ 何があればX Articlesを読むか
X 短文ポスト（X Articlesへの伏線・日常の接点・信頼の積み上げ）
```

### コンテンツ設計の正しい実行順序

逆算で出てきた「今日の行動」は短文ポストだが、**設計の順序は逆**。

| 設計ステップ | やること | 理由 |
|------------|---------|------|
| 1 | **商品・思想の深掘り**（何を売るか・何が核心か） | すべてのコンテンツの源泉。ここがブレると全部ズレる |
| 2 | **note記事を設計する**（テーマ・構成・伝えたいこと・CTA） | note記事がX Articlesと短文ポストの素材になる |
| 3 | **X Articlesを設計する**（note記事の核を切り出して予告） | note記事の内容が決まって初めてArticlesのテーマが決まる |
| 4 | **短文ポストを設計する**（X Articlesへの布石・伏線） | X Articlesの内容が決まって初めてポストのテーマが決まる |

**短文ポストは最後に決まるもの。先に決めるのは逆算ではない。**

## よくある失敗パターンと回避法

### 失敗1: 「今日できること」から考え始める

- 症状: 「今日Xに何を投稿しようか」から考える
- 問題: 投稿がnote記事・X Articlesの文脈と切断されている
- 回避: 「今日の投稿は何のX Articleの布石か？」を先に確認する

### 失敗2: 手段を先に決めてゴールを後付けする

- 症状: 「Xを毎日投稿しよう」→「目的は認知拡大」という順序
- 問題: 何のための認知かが曖昧なまま施策が走る
- 回避: 必ずゴールから「なぜこの手段か」を証明できる状態にする

### 失敗3: ファネルの「流れ」と「設計順序」を混同する

- 症状: 読者の流れ（X → note → LINE → 購入）をそのまま設計の順序にする
- 問題: 読者体験の順序と設計・制作の順序は逆
- 回避: 「読者はXから入る」が事実でも、「設計はnoteから始める」を徹底する

### 失敗4: 中間目標をゴールと混同する

- 症状: 「LINE登録者を増やすことがゴール」と設定する
- 問題: LINE登録はBizBase購入への手段であって目的ではない
- 回避: 常に「それは誰のため・何のためか」を一段上のゴールから確認する

## 逆算チェックリスト

戦略・施策を立案・評価するときに使う確認項目。

```
□ ゴールは具体的・測定可能・期限付きで定義されているか？
□ 各施策は「ゴールの直前に必要なもの」として導出されているか？
□ 「今できること」ではなく「ゴールに必要なもの」から施策を選んでいるか？
□ コンテンツ設計は「商品・思想 → note → X Articles → 短文ポスト」の順になっているか？
□ 短文ポストのテーマは、対応するX Articlesのテーマが決まった後に設計されているか？
□ 全施策をゴールとつなぐ「なぜこれが必要か」の説明ができるか？
□ 手段（ツール・媒体・形式）より先に「何を伝えるか」が決まっているか？
```

## 戦略に階層構造を持たせる

逆算で出てきた要素は必ず階層を持つ。階層を意識することでどこから手をつけるべきかが明確になる。

| 階層 | 内容 | 設計タイミング |
|------|------|--------------|
| Level 1（最上位）| 事業ゴール・ビジネスモデル | 最初に定義。変えない |
| Level 2 | チャネル戦略・ファネル設計 | Level 1から導出 |
| Level 3 | コンテンツ設計（note・Articles・LP） | Level 2から導出 |
| Level 4（最下位）| 日々の実行（短文ポスト・DM対応など） | Level 3から導出 |

上位レイヤーが決まっていない状態で下位レイヤーを設計しない。

## このスキルの参照タイミング

- 戦略ドキュメントを作成・レビューするとき（必ず参照）
- note記事・X Articles・短文ポストの設計順序を決めるとき
- 「これは本当に逆算か？」と疑問に思ったとき
- 施策の優先順位を議論するとき
