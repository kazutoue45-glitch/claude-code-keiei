# Notion データマップ

Notionに格納されている主要リソースの所在と用途。
詳細なスキーマは `.agents/skills/notion-crud/SKILL.md` を参照。

## DB一覧

| DB名 | 主な格納内容 | よく使う場面 | AI書き込み |
|---|---|---|---|
| DB_タスク管理 | タスク（ステータス・期限・PJ紐付け） | 作業の確認・進捗管理 | 指示時のみ |
| DB_プロジェクト管理 | 案件・PJ（売上目標・期間・メンバー） | 案件情報の参照 | 指示時のみ |
| DB_AI作業ログ | AI作業の記録・意思決定ログ | 作業完了後の記録 | 自動記録OK |
| DB_AI参照ナレッジ | AIスキル・マニュアル・自社商品情報 | 作業開始時の参照 | 指示時のみ |
| DB_ナレッジベース | PJ資料・記事・外部コンテンツ | リサーチ・記事確認 | 指示時のみ |
| DB_活動ログ・MTG管理 | MTG・商談・作業ログ | 商談内容の振り返り | 指示時のみ |
| Cursorアクティブコンテキスト | 作業状態の保存・復元 | save/resume | 保存/復元時 |
| Sparkie Posts | Xポスト管理・予約投稿 | ポスト作成・登録 | 指示時のみ |

## DB_AI参照ナレッジ と DB_ナレッジベース の違い

| | DB_AI参照ナレッジ | DB_ナレッジベース |
|---|---|---|
| 対象 | AIが参照する資産 | 人間が参照する資産 |
| 格納するもの | AIスキル、プロンプト、マニュアル、自社商品情報 | PJ資料、記事、外部コンテンツ、調査レポート |
| 「ナレッジベースに保存して」→ | ここではない | ここに保存する |

## リレーション構造

```
DB_ゴール設定 → DB_プロジェクト管理 → DB_タスク管理
                    ↑↓                      ↑↓
             DB_活動ログ・MTG          DB_AI作業ログ
                                            ↓
                                  DB_AI参照ナレッジ
```

## Notionの基本原則
- プロジェクト固有の情報はすべてNotionに格納する
- ローカルには行動ルール（CLAUDE.md / スキル）のみ
- 作業開始時は DB_AI参照ナレッジ を最優先で参照し、関連スキル・マニュアルがないか確認する
